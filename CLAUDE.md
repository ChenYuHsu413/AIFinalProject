# CLAUDE.md — FinalProject 專案專屬規則

> 本檔為「AI 伺服馬達故障風險預測與預測性維護建議系統」的**專案專屬 agents 規則**，
> 與上層 `../../CLAUDE.md`（通用行為準則）疊加，不重複其內容，只列本專案的硬性規定。

## 1. 文檔改動一律附日期戳

對 `docs/`、`outputs/reports/`、`README.md` 等任何**文檔**做修改或新增時，**必須附上日期戳**。

- **格式統一**：在標題或相關小節下方放一行
  `> **狀態（YYYY-MM-DD）**：<這次更新做了什麼 / 目前完成度>`。
  既有狀態戳則**更新日期與內容**，不要堆疊多行舊戳。
- 日期用**當天實際日期**（不要沿用範例裡的日期）。
- 純錯字、排版微調可省略；但凡涉及**狀態、結論、數據、完成度**變動，一律補戳。

## 2. 大型更新必須回頭翻 `docs/` 同步

任何**大型更新**——新增/移除模組、換或加資料集、改動模型方法、結果指標變動、
刪除或改名檔案——**動手前後都要翻一遍 `docs/`**，確認相關文件是否需要同步，
需要就一起改並依規則 1 補日期戳。

重點同步對象（改動時優先檢查）：

| 主題 | 對應文件 |
| --- | --- |
| Module B 主線（IMS / FPT / RUL 外推） | `docs/MODULE_B_IMS_PLAN.md`、`docs/MODULE_B_RESULTS.md` |
| 深度學習對照（1D-CNN AE） | `docs/MODULE_B_DL_PLAN.md` |
| 外部資料集評估與擴充 | `docs/DATASET_EVALUATION.md` |
| 多軌跡泛化（XJTU Module B+） | `docs/MODULE_B_PLUS_XJTU_PLAN.md` |
| 報告大綱 / 工作紀錄 | `outputs/reports/` |

- 文件之間以相對連結互相引用；新增文件時補上交叉連結。
- 發現文件內容與程式碼或實際產物**不一致**時，先指出落差，再依規則 1 更新，不要默默改寫。

## 3. 誠實性紅線（報告防禦用，務必遵守）

- 不得把 **AI4I 2020** 說成真實伺服馬達資料（它是合成資料）。
- 不得宣稱 **IMS Set 2** 單軌跡結果可泛化到其他軸承/馬達。
- 不做單軌跡資料上的深度 RUL 回歸（會撞外推牆）。
- **ESP32** 定位為未來實場接入 / IoT demo，非現階段訓練資料來源。
