## X.1 動機：現有資料的兩個結構性限制

本系統目前建立於兩份資料之上，各有明確且已知的限制：

- **AI4I 2020（Module A）**：為**合成**資料，用於展示靜態製程風險分類的完整 CRISP-DM
  流程與模型治理（10 模型 × 5 特徵組、F1/Recall/ROC-AUC/PR-AUC、SHAP、防資料洩漏）。
  **它不代表真實伺服馬達的物理行為。**
- **IMS Set 2（Module B）**：為**真實** run-to-failure 振動資料，但**僅含單一退化軌跡
  （B1 外圈故障）**。因此所建立之健康指標與告警門檻**無法進行跨設備/跨軸承的泛化驗證**。

換言之，本系統目前**缺乏「多軌跡泛化證據」**與**「真正馬達電氣訊號」**。本文件評估公開
資料集，規劃在不發散專案的前提下補強之路線。

## X.3 推薦與取捨：為何優先 XJTU-SY

在六個候選中，**XJTU-SY 與本專案契合度最高**：

1. 它與 IMS 同為振動 run-to-failure 資料，**可直接沿用既有的振動特徵抽取
   （`src/features/vibration_features.py`）與趨勢外推（`src/models/rul_extrapolation.py`）
   管線**，邊際工程成本最低。
2. 它以**多條獨立退化軌跡（3 工況 × 5 軸承）**，直接補足 IMS Set 2 單軌跡**無法進行
   跨設備泛化驗證**的結構性限制——這是本系統目前最關鍵的缺口。
3. **Paderborn** 提供馬達定子電流（MCSA 非侵入式診斷），最貼近「馬達」主題，列為
   **次優先與未來電流診斷擴充**。
4. **CWRU** 因屬人工 EDM 故障且基準已飽和，僅作為分類教學對照，不納入主線；
   **PMSM 溫度**屬溫度回歸而非故障預測，列為未來「熱風險估測」加值方向；
   **FEMTO** 因失效模式雜、RUL 誤差天生偏大，作為多軌跡 RUL 的延伸對照而非首選。

## X.4 限制與誠實聲明

- AI4I 2020 為**合成**資料，用於展示流程與方法治理，**不代表真實伺服馬達行為**。
- IMS Set 2 為**單一退化軌跡**，所得健康指標與門檻**不宣稱可跨軸承/跨設備泛化**。
- 本專案以「**旋轉機械／電動機預測性維護**」為方法範疇，**以伺服馬達為應用情境**；
  目前公開資料中缺乏伺服馬達專屬 run-to-failure 資料，此為領域共通限制，本研究如實
  揭露而非迴避。

## X.5 從公開資料集到實場資料的遷移路線

```
公開 benchmark            跨設備驗證              多感測器融合           實場域 IoT 接入
IMS（單軌跡） ──▶ XJTU-SY（多軌跡/多工況） ──▶ Paderborn（電流+振動+溫度） ──▶ ESP32 + GCP
  已完成              已完成（Module B+）       已完成（Module C · MVP）       推遲
```

**未來工作（已排序）**：(1) Paderborn 模組 C **MVP 已完成**（馬達電流 MCSA + 振動分類、人工→真實泛化），
延伸軌 **CE1 領域自適應 + CE4 FastAPI 端點已完成**；**CE2 MCSA 頻譜邊帶 / CE3 全 4 工況不納入本專案（2026-06-29 決定不做）**。
(2) 其餘皆推遲（FEMTO / Mendeley / PMSM、ESP32）；**模組 B 1D-CNN Autoencoder 已確定不做**（DL 已由 Servo 主線達成）。

**ESP32 補充**：ESP32 定位為**實場資料接入／IoT 展示**，非現階段訓練資料來源。需誠實
指出：ESP32＋MEMS 加速度計的取樣率（單軸約數 kHz）遠低於 IMS 的 20 kHz×4，無法進行
實驗室級頻譜診斷，僅適合粗趨勢／溫度監測——此「實驗室 → 實場域」之取樣率與雜訊落差
（domain shift），正是上述遷移路線最後一棒所要面對的核心挑戰。
