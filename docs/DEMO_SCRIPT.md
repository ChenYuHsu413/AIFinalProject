## 1. Overview 監控首頁（約 1.5 分）

**點**：首頁 `/`。
**講**：「這是指揮中心首頁。上方 KPI——監控設備、平均健康、高風險、作用中告警。」
「中間這排設備卡（Servo-A01/A02/A03/TestBench）的健康分數、風險、異常特徵，**是後端用真實參考模型
即時算出來的**（`/servo/fleet`），不是寫死的。下方是健康趨勢圖與告警。」
**預期**：機群卡顯示不同健康（如 A01 94 綠、TestBench 11 紅），告警列表有資料。
**亮點**：「機群與告警都由真模型在 demo 運轉段上衍生——這是把 ML 結果接成監控介面的展示。」

## 2. 設備詳情 + 真預測（約 1 分）

**點**：點任一張設備卡 → 進 `/equipment/[id]`。
**講**：「點進單一設備：上半是健康快照與感測器趨勢；**下半『參考模型預測』是即時打後端

## 3. Servo 健康儀表板（約 1 分）

**點**：側欄「Servo 健康儀表板」。選一筆運轉段 → 估測。
**講**：「選一段運轉資料，模型輸出健康狀態 LN/LO/MED/HI、退化分數 DV、風險、信心，
還有環形健康分數計。分類與 DV 矛盾時會跳一致性警告。」
**預期**：健康分數環 + 風險標籤 + 各狀態機率 + 建議處置。

## 4. AI 訓練模擬器（教學亮點，約 1.5 分）

**點**：側欄「AI 訓練模擬器」。選資料量 / 特徵組 / 演算法 → 開始訓練。
**講**：「這頁是教學型的：使用者可以選資料量、特徵組、演算法，**後端即時訓練一個小模型（<0.4 秒）**，
跟離線的 Reference Model 對照。可以現場示範『資料量越多、特徵越貼切，越接近參考模型』。」
**預期**：訓練時間、Accuracy/F1、混淆矩陣、小模型 vs Reference 對照。
**操作建議**：先用 100 筆跑一次（差），再用 5000 筆跑一次（接近 reference）→ 對比講效果。

## 6. 模組 C · Paderborn（誠實性頭條，約 1.5 分）★最重要

**點**：側欄 Legacy → 模組 C。
**講**：「這是我認為最有價值的一頁。用**真實馬達定子電流（MCSA）**做故障分類。
**Baseline macro-F1 = 1.00**，看起來完美——**但這是『人工製造的故障』訓練、測同分佈**。
**換成真實加速壽命損傷後，macro-F1 掉到 0.20**，真實損傷幾乎全被誤判（看右邊混淆矩陣對角線潰散）。
這就是工業 ML 最常見的陷阱：**人工故障訓練無法直接泛化到真實劣化**。我們**如實呈現這個落差、
不只報好看的 baseline**。」
**預期**：兩張混淆矩陣（baseline 漂亮 vs 真實潰散）+ gap 0.80。

**接著往下捲到「CE1 領域自適應」**（同一頁，約再 1 分）：
**講**：「但我們不只是觀察失敗——我們**試著修，並診斷為什麼修不動**。
①**兩種無監督特徵對齊**（CORAL / 工況感知標準化，只用目標未標註特徵）**完全沒用**，
CORAL 甚至更糟（0.04）。②**機制診斷**回答為什麼：我們量化每個特徵在人工 vs 真實的分布位移，
和 baseline 對它的倚重程度——**Spearman = 0.50**，也就是**模型最依賴的判別特徵，正是位移最大的那些**
（看散點圖右上角）。判別軸本身被破壞，所以線性對齊救不回。③**唯一有效的是 few-shot**：
每類只要 10 筆真實標籤，macro-F1 就從 0.20 拉到 0.54、40 筆到 0.65——我們**量化了「要多少真實標籤才夠」**。
連『先 CORAL 再 few-shot』都更差，完全符合診斷。」
**預期**：無監督長條（皆 ≤ baseline）+ few-shot 學習曲線（兩條，純 few-shot 在上）+ 診斷散點（正斜率）。
**一句話收**：「這把一個被動的『泛化失敗』，變成**有診斷、有量化、有可操作結論**的研究發現。」

## 9. 預期會被問的問題（先想好答案）

| 問題 | 回答要點 |
| --- | --- |
| 「這是真的伺服馬達資料嗎？」 | 主線 Servo 用真實 **PHM FMCRD** 資料集（高擬真**模擬**、非工廠遙測）；A 是合成；B/B+ 是軸承試驗台；C 是 PMSM 試驗台。如實標示。 |
| 「FMCRD 算真實資料嗎？」 | 「真實」指完整大型公開 PHM 資料集本身（相對於先前 placeholder 合成）；它是高擬真**模擬**、非真實工廠遙測，我們不誇大為產線資料。 |
| 「macro-F1 0.20 不是很差嗎？」 | 那正是重點——揭露人工→真實泛化落差，比只報 baseline 1.0 誠實且有價值。且我們以 CE1 進一步**診斷並部分修復**（few-shot 10/40 筆→0.54/0.65）。 |
| 「有沒有試著修這個落差？」 | 有，CE1：兩種無監督對齊（CORAL/z-score）無效、機制診斷 Spearman 0.50 證明判別軸位移最大、唯 few-shot 有效並量化所需真實標籤數。 |
| 「為什麼 CORAL 沒用、甚至更糟？」 | CORAL 只能對齊協方差（線性仿射）。我們的診斷顯示判別特徵本身位移最大，判別軸被破壞，線性對齊無法復原；跨 reg 0.01–100 掃描全 < baseline，非調參問題。 |
| 「few-shot 用了真實標籤，還算泛化嗎？」 | 如實揭露屬 **few-shot 非零樣本**；價值在量化「每類幾筆真實標籤能換多少效能」（10→0.54、40→0.65），這是可操作的部署指引。 |
| 「能預測剩餘壽命嗎？」 | 健康監測可泛化；絕對 RUL 跨工況受限，我們不誇大。 |
| 「會控制馬達嗎？」 | 不會。定位是決策輔助，不接控制迴路。 |
| 「有用到深度學習嗎？」 | 有，兩階段真 PyTorch：Phase A MLP + 神經 autoencoder（特徵）、Phase B 真 1D-CNN（原始波形）。見 §11。 |
| 「為什麼 1D-CNN 沒比聚合特徵好？」 | 退化訊號在長時統計裡，小型 per-run 模擬資料下深度模型不占優；而我們**如實報告**、不挑好看的 seed——這正是專案的誠實性。 |

## 11. 深度學習・一分鐘版（上台講稿）

> **狀態（2026-06-28）**：深度學習兩階段的濃縮口頭/投影片版；完整說明見
> [`FINAL_REPORT.md` §9.1](FINAL_REPORT.md)。

**一句話**：兩階段真・PyTorch 深度學習——先在聚合特徵上做 MLP + 神經 autoencoder，再在**原始波形**上
做真正的 1D-CNN；能直接從訊號學到退化，但我們誠實呈現它（macro-F1 0.69）仍略低於聚合主線（0.76），不誇大。

**投影片 4 點**
- **Phase A**：PyTorch MLP（分類 + DV 回歸）+ 神經 autoencoder（健康資料擬合、重建誤差隨退化上升）——取代先前用 PCA 充當的假 autoencoder。
- **Phase B**：真 **1D-CNN** 直接吃原始 FMCRD 波形（每段 run 能量包絡，8 通道 × 256，從 22 GB zip 串流抽取）。
- **誠實工程**：發現「每短窗分類」上限只 ~0.36 → 改「每段 run」才有效；多 seed 驗證後把樂觀的 0.73 **如實下修到 0.69（±0.03）**。
- **架構**：離線訓練、雲端只讀結果 JSON、映像不裝 torch；後端 `/servo/cnn_results`、報表頁可視化。

**口頭版（~30 秒）**
> 「深度學習分兩階段：第一階段在工程特徵上做 PyTorch 的 MLP 加神經 autoencoder；第二階段是重點——真正的
> 1D-CNN，直接吃原始伺服馬達波形，我們從 22GB 原始 zip 串流抽出每段運轉的能量包絡餵 CNN。過程中有個誠實的
> 發現：單一短窗根本分不出來、要用整段 run 才行；而且小資料下分數對隨機種子敏感，所以我們做了多種子驗證、
> 把一開始好看的 0.73 下修成穩健的 0.69。結論是——CNN 確實能從原始波形學到退化，但這份模擬資料上還是聚合
> 特徵主線 0.76 最強，我們不灌水。」

**防禦金句**（被問「為什麼 CNN 沒比較好」）
> 「退化訊號在長時統計裡，短資料的深度模型不占優——而我們選擇如實報告，而不是挑一個好看的 seed。這正是這個專案的誠實性。」
