# 模組 B 進階規劃 — 深度學習對照（1D-CNN Autoencoder 異常偵測）

> **定位**：在模組 B 既有的「健康度 / RUL 趨勢外推」之上，加一條**深度學習**對照軌，
> 作為期末報告的進階加分與方法亮點。
>
> **目標已定案**：純加分 / 展示 DL 能力 —— 準確率其次，重點是**方法亮點 + 誠實**。
>
> **核心決策**：做 **1D-CNN Autoencoder 異常偵測**，**不**做 DL 的 RUL 回歸，也**不**做健康階段分類。
>
> 本文件只做分析與規劃，尚未實作。對照主線規劃見 [`MODULE_B_IMS_PLAN.md`](MODULE_B_IMS_PLAN.md)、
> 實測結果見 [`MODULE_B_RESULTS.md`](MODULE_B_RESULTS.md)。

## 0. 一句話總結

> 在 IMS Set 2 這份**只有 B1 一條 run-to-failure 軌跡**的資料上，能誠實展示深度學習、
> 又不撞「單軌跡無法外推」那道牆的做法，是**只用早期健康段訓練 1D-CNN Autoencoder**，
> 用重建誤差上升偵測退化，並與既有的統計 FPT 互相印證。

## 1. 分析：為什麼「進階模型」不能照原規劃做 RUL 回歸

原 `MODULE_B_IMS_PLAN.md` 第 6 步把進階模型寫成「1D-CNN / LSTM 做 RUL 回歸對照」。
**有了真實數據後，這條路已被證實是死的**，理由分兩層：

### 1.1 主線已經實測撞過同一道牆

| 做法 | 結果 | 根因 |
| --- | --- | --- |
| 監督式回歸（RF / GBM）直接預測 RUL 小時數 | **MAE ≈ 120 h、R² ≈ −76（爛掉）** | 測試段 RUL 範圍（0–48.5 h）完全落在訓練段（50–164 h）之外，樹模型**無法外推**單調目標 |
| 改走趨勢外推法（PHM 標準，最終採用） | 退化區 MAE ≈ 25 h、提前量 3.1 天 | 不讓模型外推絕對小時數，改擬合退化趨勢反推 RUL |

### 1.2 深度學習會撞**同一道牆，而且更嚴重**

- 問題的本質不是模型不夠強，是**任務框架**：要「預測未來還剩幾小時」就得外推到訓練時沒見過的尾端。
- **IMS Set 2 只有 B1 一條退化軌跡**。深度模型（1D-CNN / LSTM）比樹模型**更吃資料、更需要多條軌跡**才能學到可泛化的退化規律。
- 用單一軌跡訓練一個 LSTM 做 RUL 回歸 → 必然嚴重過擬合 + 無法外推，且在報告裡是明顯破綻：
  「你用一條軌跡訓練深度回歸模型？」一句就破功。

> **結論**：DL 做 RUL 回歸，在這份資料上不可行，且不誠實。捨棄。

## 2. 決策：在單軌跡資料上「誠實又夠 DL」的做法比較

| 路線 | 任務 | 單軌跡可行？ | 誠實風險 | 加分展示度 | 取捨 |
| --- | --- | --- | --- | --- | --- |
| RUL 回歸（原規劃） | 預測剩餘小時數 | ❌ 撞外推牆 | 高（破綻） | — | **捨棄** |
| 路線 1：健康階段分類 | 1D-CNN 分「健康/退化/嚴重」 | △ 有驗證陷阱 | 中 | 高（有準確率數字） | 不選，見 §2.1 |
| **路線 2：Autoencoder 異常偵測** | 1D-CNN AE，重建誤差升 = 退化 | ✅ 標準做法 | **低** | 高（夠 DL 且故事連貫） | **採用** |

### 2.1 為什麼不選路線 1（健康階段分類）

- 健康 / 退化 / 嚴重是**三段連續時間塊**。單軌跡下：
  - **時間切分**會把整段拿去訓練或測試（某些類別在測試段根本不存在）→ 破掉。
  - **隨機切分**因相鄰快照幾乎相同 → 資料洩漏，準確率虛高。
- 要做乾淨，得拉 IMS Set 1 / Set 3 的軸承進來做**跨軸承切分** → 等於再做約 3 倍資料工程，對「純加分」CP 值低，且更容易被問倒。

### 2.2 為什麼採用路線 2（Autoencoder 異常偵測）

1. **單軌跡上唯一不撞牆、又不露破綻的 DL 做法**：只拿**早期健康段**訓練（無監督），學「正常振動長什麼樣」；之後重建誤差隨退化上升，穿越門檻 = 告警。不需外推、不需標籤。這正是工業界對單一 run-to-failure 做健康監測的標準手法。
2. **故事線跟既有成果完美接上**：重建誤差曲線會與主線已做出的**統計 FPT 退化起點**互相印證 → 報告敘事變成「統計 FPT + 深度 Autoencoder 兩種獨立方法都偵測到同一個退化起點」。
3. **視覺好賣**：重建誤差爬升 → 穿越門檻 → 提前量，與既有健康曲線同一個畫面語言。
4. **夠 DL**：用 **1D-CNN autoencoder**（卷積編碼器 + 解碼器吃原始 20480 波形），一次打勾「卷積網路」+「深度學習」+「無監督表徵學習」。

## 3. 工程取捨：雲端依賴（重要）

DL 會引入 **PyTorch**（Streamlit Cloud build 多約 1–2 GB，會顯著拖慢部署）。

**決策：沿用現有「離線訓練 + commit 小產物」模式**（與 parquet / joblib 一致）：

- 深度模型**只在本機離線訓練**。
- 只把結果（重建誤差曲線、門檻、提前量）存成小 CSV / JSON 並 commit。
- Dashboard **只讀結果畫圖**，**`torch` 不進 `requirements.txt`** → 雲端 demo 不受影響、不變慢。

> 雲端只呈現結果，不在雲端跑訓練/推論。

## 5. 各階段細節

### ① 訓練集切分 — 只取「早期健康段」
- 用主線已偵測到的**統計 FPT** 之前的快照當健康段（或保守取前 ~20% 壽命）。
- 只用 **B1 通道**（與主線建模對象一致）。
- 關鍵誠實點：**測試/評估段（退化後）絕不進訓練集** —— AE 只見過健康，才有「對退化感到陌生」的偵測力。

### ② 模型 — 1D-CNN Autoencoder
- 輸入：原始波形片段（如每檔 20480 取樣，或降採樣 / 切窗以控制參數量）。
- 編碼器：數層 1D 卷積 + 下採樣 → 低維潛在表徵；解碼器對稱還原。
- 損失：重建 MSE。輕量即可（這是展示，不是 SOTA 競賽）。

### ③ 健康指標 — 重建誤差
- 對全段每檔算重建誤差 → 平滑（與主線健康指標同一套平滑慣例）。
- 門檻 = 健康段重建誤差 `mean + n·std`（n 寫進 config）。
- **AE-FPT** = 重建誤差首次穩定穿越門檻的時間點。

### ④ 與統計 FPT 印證（報告核心賣點）
- 並列：統計 FPT 提前量（已知 3.1 天） vs AE-FPT 提前量。
- 敘事：兩種**獨立方法**（統計 / 深度無監督）指向同一退化起點 → 結論更可信。

### ⑤ Dashboard 呈現
- 在既有「動態健康度」區新增「重建誤差曲線」：誤差爬升 + 門檻線 + AE-FPT 穿越點。
- 純讀 CSV/JSON，**不引入 torch**，雲端可正常顯示。

## 6. config / 測試 / 邊界

- `config.yaml` 在 `ims:` 下新增 `autoencoder:` 子區段：健康段切分依據、視窗/降採樣、潛在維度、訓練 epoch、門檻 `n_std`、輸出路徑。
- `tests/`（合成資料，不需 torch 重訓、不需真資料）：
  - 重建誤差工具：對「健康段」誤差低、對「注入脈衝的退化段」誤差升高；
  - 門檻與 AE-FPT 邏輯：穿越點偵測正確、提前量非負；
  - 若測模型前向，用極小張量驗證 shape，不驗收斂。
- **完全不改動 AI4I 軌，也不改動模組 B 既有的趨勢外推主線**；本模組純疊加。

## 8. 報告故事線

> 模組 B 用**兩種獨立方法**偵測同一退化起點：
> 統計健康指標 FPT（提前 3.1 天）＋ 深度 1D-CNN Autoencoder 重建誤差，
> 互相印證，並誠實說明：在**單一退化軌跡**上不做深度 RUL 回歸（會撞外推牆），
> 改以無監督異常偵測展示深度學習能力。

**誠實聲明（務必寫進報告）**：
- 本模組為**單軌跡無監督異常偵測**，非多軌跡泛化模型；不宣稱可跨軸承泛化。
- 深度 RUL 回歸經評估後**刻意不做**，理由為單軌跡無法外推（見 §1）。
- 多軌跡 DL（拉 Set 1 / Set 3）與 Paderborn / MCSA 電流診斷，列為未來工作。
