  ========================                                                      
  二、資料集說明                                                                
  =======                                                                       
                                                                                
  請使用 UCI AI4I 2020 Predictive Maintenance Dataset。                         
                                                                                
  資料集欄位大致包含：                                                          
                                                                                
  ID 類欄位：                                                                   
                                                                                
  * UDI                                                                         
  * Product ID                                                                  
                                                                                
  可用特徵欄位：                                                                
                                                                                
  * Type                                                                        
  * Air temperature [K]                                                         
  * Process temperature [K]                                                     
  * Rotational speed [rpm]                                                      
  * Torque [Nm]                                                                 
  * Tool wear [min]                                                             
                                                                                
  目標欄位：                                                                    
                                                                                
  * Machine failure                                                             
  * TWF                                                                         
  * HDF                                                                         
  * PWF                                                                         
  * OSF                                                                         
  * RNF                                                                         
                                                                                
  請注意：                                                                      
                                                                                
  1. UDI 與 Product ID 通常不應作為模型特徵。                                   
  2. Machine failure 是第一階段主要預測目標。                                   
  3. TWF、HDF、PWF、OSF、RNF 是故障類型標籤，不能直接拿來預測 Machine           
  failure，否則會造成 data leakage。                                            
  4. 如果要使用 TWF、HDF、PWF、OSF、RNF，請設計成第二階段故障類型分析，或在報告 
  中清楚說明其用途。                                                            
  5. 報告中必須清楚說明 AI4I 是合成資料，不是真實伺服馬達資料。

  - Five engineered features (temp_diff, power_proxy, wear_torque_interaction,

  - Five feature sets compared: A baseline / B engineered / C SelectKBest / D
  RFE / E RF-importance.
  - Models compared: LR + DT + RF + SVM + GB + KNN + MLP + NB, plus XGBoost &
  LightGBM if installable (model_registry gracefully skips missing libs).
  - Best-model criterion: F1 (configurable), with all six metrics reported (acc

      331 +## 14. 模型選擇邏輯                                           
      332  
      355 -- **Why not just Accuracy?** With ~3 % positives, predicting "
          -no failure"                                                   
      356 -  always gives ~97 % accuracy yet is useless. We therefore rep
          -ort                                                           
      357 -  Precision, Recall, F1, ROC-AUC and PR-AUC and pick the best 
          -model on                                                      
      358 -  F1 (configurable via `config.yaml::modeling.scoring_for_best
          -`).                                                           
      359 -- **Recall vs Precision trade-off.** In predictive maintenance
          -, missing a                                                   
      360 -  true failure (false negative) typically costs more than insp
          -ecting a                                                      
      361 -  healthy machine (false positive). The selection logic and th
          -e                                                             
      362 -  threshold can be tuned to bias toward recall once a real cos
          -t model is                                                    
      363 -  available.                                                  
      364 -- **Class imbalance handling.** Where the estimator supports i
          -t, we use                                                     
      365 -  `class_weight="balanced"`.                                  
      366 -- **Leakage avoidance.** All scaling and encoding live inside 
          -a                                                             
      367 -  `Pipeline` that is fit only on the training fold. The five f
          -ailure-type                                                   
      368 -  columns are dropped from `X` before training.               
      333 +- **為什麼不只看 Accuracy？** 正樣本約 3%，模型只要全部猜「沒 
          +故障」就有 ~97% Accuracy                                      
      334 +  但完全沒有實用價值。因此本專案以 Precision / **Recall** / F1
          + / ROC-AUC / **PR-AUC**                                       
      335 +  共同評估，預設以 F1 挑選最佳模型（可於 `config.yaml::modelin
          +g.scoring_for_best`                                           
      336 +  調整）。                                                    
      337 +- **Recall 與 Precision 的取捨。** 在預測性維護情境，漏報（fal
          +se negative）成本                                             
      338 +  通常高於誤報（false positive）；若取得實際成本模型後，可在 `
          +predict.py` 中調整                                            
      339 +  決策門檻使其更偏向 Recall。                                 
      340 +- **不平衡處理。** 支援 `class_weight="balanced"` 的模型都已啟
          +用該設定。                                                    
      341 +- **避免 data leakage。** 所有 scaling 與 encoding 都包在 `Pip
          +eline` 內，僅在訓練                                           
      342 +  折上 fit；五個故障類型欄位在訓練前即從 X 移除。             
      343

  方案 B — 用 curl + GitHub Personal Access Token（如果你想自動化）

  需要先去 https://github.com/settings/tokens 產生一個有 repo 權限的
  token，然後在 PowerShell 跑：

  $TOKEN = "ghp_你的token"
  $desc = "以 UCI AI4I 2020 建立的預測性維護原型：CRISP-DM 全流程、10 
  模型比較、Optuna 調參、SHAP 解釋、Streamlit + FastAPI + Docker + CI"

  # 設 description
  curl.exe -X PATCH `
    -H "Authorization: token $TOKEN" `
    -H "Accept: application/vnd.github+json" `
    https://api.github.com/repos/ChenYuHsu413/AIFinalProject `
    -d (@{description=$desc} | ConvertTo-Json)

  # 設 topics
  curl.exe -X PUT `
    -H "Authorization: token $TOKEN" `
    -H "Accept: application/vnd.github+json" `
    https://api.github.com/repos/ChenYuHsu413/AIFinalProject/topics `
    -d '{"names":["predictive-maintenance","machine-learning","crisp-dm","ai4i-2
  020","streamlit","fastapi","shap","optuna","python","docker"]}'
