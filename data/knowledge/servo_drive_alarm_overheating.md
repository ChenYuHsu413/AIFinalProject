# 伺服驅動器告警與馬達過熱（Drive Alarm / Overheating）

> 離線維修知識庫種子內容。一般性參考；實際告警碼定義以各廠牌驅動器手冊為準。

## 常見伺服驅動器告警類型（通用說明）
不同廠牌告警碼不同，但常見類別包括：
- 過電流（overcurrent）：瞬時電流超限，常與卡滯 / 短路 / 加減速過急有關。
- 過載（overload）：長時間高扭矩輸出累積熱量。
- 過電壓 / 欠電壓（over/under voltage）：再生能量過大或供電不穩。
- 位置誤差過大（excessive position error / following error）：追隨能力不足。
- 編碼器異常（encoder error）：回授訊號遺失或雜訊。
- 過熱（overheat）：驅動器或馬達溫度超限。

> 查詢實際告警碼時，請對照該驅動器型號的官方手冊；本知識庫不臆測特定型號代碼。

## 馬達過熱（overheating）
### 可能原因
- 持續過載 / 過電流。
- 散熱不良（風扇故障、散熱片積塵、環境溫度過高）。
- 潤滑不足造成摩擦發熱。
- 切換頻率 / 參數設定不當。

### 建議檢查項目
1. 確認負載與工作週期（duty cycle）是否超出規格。
2. 檢查散熱風扇、散熱片與環境通風。
3. 量測馬達溫度趨勢，對照電流 / 扭矩變化。
4. 排除潤滑不足造成的額外摩擦熱。

## 處置原則
- 告警發生時優先確保安全，必要時停機。
- 記錄告警時間、當下負載 / 電流 / 溫度，供後續分析。
- 重大或重複告警應通報維護單位並由合格人員處理。
